# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
  - リポジトリ: ファイルやフォルダの状態を保管しておく場所
  - リモートリポジトリ: インターネット上で複数人で共有するリポジトリ
  - ローカルリポジトリ: ローカル環境上に存在するリポジトリ

## プッシュとマージの違いは何でしょうか？
  - プッシュ: ローカルリポジトリ上のコミットをリモートリポジトリにUPすること
  - マージ: あるブランチのコミットをmainブランチなどに結合すること

## コミットとプッシュの違い
  - コミット: ローカルリポジトリに変更履歴を保存すること、そのデータ
  - プッシュ: コミットをリモートリポジトリに共有すること

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
  - 行った変更を正確に表すように書く。
  - 各チームで記述ルールがある場合は、そのルールに従って書く

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
  - ローカルでマージする場合、ブランチに変更内容をコミットした後、ローカルリポジトリの幹ブランチにマージする
  - プルリクエストでマージする場合、ローカルリポジトリのブランチをリモートリポジトリにプッシュした後、リモートリポジトリでプルリクエストを作成し、承認者によって承認されるとリモートリポジトリの幹ブランチにマージされる

## コンフリクトを起こしてしまった場合、どう対処すべきですか？
  - 状況に応じて３通りから選択する
    - 先の変更内容を取り込む
    - 後に変更したい内容を取り込む
    - 両方ともの変更内容を取り込む
  - 先の変更内容の意図が読めない場合は、変更した人とすりあわせて適切に選択する。
