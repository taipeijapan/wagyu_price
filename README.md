# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
リモートリポジトリ：クラウド上に存在し、チームメンバーの変更内容を集約するリポジトリ
ローカルリポジトリ：ローカル環境に存在し、リモートリポジトリを反映させた内容から、そのローカル環境で変更した内容を保存するリポジトリ


## プッシュとマージの違いは何でしょうか？
プッシュ：ローカルリポジトリでコミットした内容を、リモートリポジトリに反映させること
マージ：ブランチの変更内容を、幹ブランチに反映させること


## コミットとプッシュの違い
変更内容をローカルリポジトリに反映させるか、リモートリポジトリに反映させるか

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
変更内容がわかるよう、簡潔に書く

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
ローカルでマージする際はローカル内でそのままマージされるが、
プルリクエストでマージする場合、リモートリポジトリにて承認者の承認によってマージされる。


## コンフリクトを起こしてしまった場合、どう対処すべきですか？
同じ部分を変更した担当者と話し合い、どちらの内容を優先するか、もしくは両方の変更を反映させるか決定する。
